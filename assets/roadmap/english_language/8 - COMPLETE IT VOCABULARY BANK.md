### Core Development Terms (Must Know)

**Software Development Lifecycle**:

1. Requirements - what the software should do
2. Design - how the software will work
3. Development - writing the code
4. Testing - checking if it works correctly
5. Deployment - putting it on the server
6. Maintenance - fixing issues and updates

**Common Development Phrases**:

- "I'm developing a feature"
- "I'm fixing a bug"
- "I'm writing unit tests"
- "I'm refactoring the code"
- "I'm reviewing a pull request"
- "I'm deploying to production"
- "I'm investigating an issue"
- "I'm optimizing performance"

### Programming Concepts

**Basic Terms**:

- Variable - stores data
- Function - reusable code block
- Loop - repeats code
- Condition - if/else logic
- Array - list of items
- Object - data structure with properties
- Class - blueprint for objects
- Method - function inside a class

**Example Usage**:

- "I created a function that calculates the total"
- "This variable stores the user's name"
- "I'm using a loop to process all items"
- "I added a condition to check if the user is logged in"

### Web Development

**Frontend**:

- HTML - structure of web pages
- CSS - styling and layout
- JavaScript - interactivity
- React/Vue/Angular - frontend frameworks
- Component - reusable UI element
- State - data that changes
- Props - data passed to components
- Responsive design - works on all devices

**Backend**:

- Server - computer that responds to requests
- API - interface for communication
- Endpoint - specific URL path
- Database - where data is stored
- Query - request for data
- Route - URL path handler
- Middleware - code that runs between request and response
- Authentication - verifying user identity

**Example Usage**:

- "I'm working on the frontend using React"
- "I'm building an API with Node.js"
- "I need to query the database"
- "I'm creating a new endpoint for user registration"

### Database Terms

**Basics**:

- Table - collection of data
- Row/Record - single entry
- Column/Field - specific piece of data
- Primary Key - unique identifier
- Foreign Key - link to another table
- Index - speeds up searches
- Query - request for data
- Transaction - group of operations

**Types**:

- SQL - structured database (MySQL, PostgreSQL)
- NoSQL - flexible database (MongoDB, Redis)
- Relational - data in tables with relationships
- Document-based - data in JSON-like documents

**Example Usage**:

- "I'm writing a SQL query to get user data"
- "I added an index to improve performance"
- "I'm using MongoDB for this project"
- "I created a relationship between users and orders"

### DevOps & Infrastructure

**Key Terms**:

- Server - computer that hosts applications
- Cloud - online computing resources (AWS, Azure, Google Cloud)
- Container - packaged application (Docker)
- Deployment - moving code to production
- CI/CD - automated testing and deployment
- Version Control - tracking code changes (Git)
- Repository - code storage (GitHub, GitLab)
- Branch - separate version of code
- Merge - combining code changes
- Commit - saving changes
- Push - uploading changes
- Pull - downloading changes

**Example Usage**:

- "I pushed my changes to GitHub"
- "I'm deploying the app to AWS"
- "I created a Docker container"
- "I set up CI/CD pipeline"
- "I merged my branch into main"

### Agile & Project Management

**Key Terms**:

- Sprint - 1-2 week work period
- Backlog - list of tasks to do
- User Story - feature from user perspective
- Task - specific work item
- Standup - daily team meeting
- Retrospective - review meeting
- Scrum - agile framework
- Kanban - visual workflow board
- Epic - large feature
- Bug - code error
- Priority - importance level
- Estimate - time prediction

**Example Usage**:

- "We're starting a new sprint next week"
- "I picked up a task from the backlog"
- "This bug is high priority"
- "I estimate this will take 2 days"
- "Let's discuss this in the retrospective"

### Testing Terms

**Types of Testing**:

- Unit Test - tests individual functions
- Integration Test - tests how parts work together
- End-to-End Test - tests complete user flow
- Manual Test - human testing
- Automated Test - code that tests code
- Regression Test - ensures old features still work

**Quality Terms**:

- Bug - error in code
- Issue - problem to solve
- Debug - find and fix errors
- Code Review - checking someone's code
- Quality Assurance (QA) - ensuring quality

**Example Usage**:

- "I wrote unit tests for this function"
- "I'm debugging a problem with the login"
- "QA found a bug in the payment feature"
- "I need to fix this issue before deployment"

### Performance & Optimization

**Key Terms**:

- Performance - how fast/efficient
- Optimization - making code faster
- Caching - storing data for quick access
- Load Time - how long to load
- Response Time - how long to respond
- Scalability - handling more users
- Bottleneck - slowest part of system
- Memory Leak - wasted memory
- Latency - delay in response

**Example Usage**:

- "I optimized the database queries"
- "I added caching to improve performance"
- "We need to fix this memory leak"
- "The application scales well"

### Security Terms

**Basics**:

- Authentication - verifying identity
- Authorization - checking permissions
- Encryption - protecting data
- Password - secret for access
- Token - digital key
- Session - active user connection
- SSL/HTTPS - secure connection
- Vulnerability - security weakness
- Firewall - security protection

**Example Usage**:

- "I implemented user authentication"
- "I'm encrypting sensitive data"
- "I fixed a security vulnerability"
- "I'm using JWT tokens for authorization"
